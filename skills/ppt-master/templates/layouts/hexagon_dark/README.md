## 5 类页型

1. **封面 cover** — 全屏深青 + 主副标题 + 左上 Logo + 底部亮点标签
2. **目录 toc** — 5 章节卡片（深底 + 编号 + 描述）
3. **章节过渡 chapter** — 黑色圆角横条 + 青色编号 + 顿号 + 章节名 + 黄绿描述
4. **内容 content** — 深色径向背景 + 32pt 主标题 + 17pt 副标题（与倒梯形上沿平齐）+ 主体 + 底部金句条
5. **封底 ending** — 深青 + THANK YOU 80pt + Visit + 公司

## 三栏卡片（核心 · 强烈推荐用）

### 类型 1：痛点对比（章节2 用）
- 现状（青色 + 眼睛图标）
- 痛点（橙色 + 警告图标）
- 影响（红色 + 下降图标）
- 业务影响列用 "数字：描述" 自动拆为大数字 18pt + 小描述 13pt
- 底部橙色金句 + ⚠ 警示符

### 类型 2：PSV 解决方案（章节4 通用）
- 痛点（橙色 + 警告图标）
- 方案（青色 + 对勾图标）
- 价值（黄绿 + 上升图表图标）
- 价值列同样支持"数字：描述"自动拆分
- 底部青色金句

## 重要约束

1. **内容页背景**：layout1 自动提供深色径向渐变，**严禁**在 slide 覆盖白色矩形
2. **副标题位置**：y=686000 与 layout 自带倒梯形（image5.png）上沿平齐
3. **不放页码**：王老师明确不要 PAGE x/N
4. **图标用 PNG**：picture_xml + iconfont 风格 PNG，不用 prstGeom 自绘
5. **业务模块完整三件套**：业务流程 + 功能架构 + PSV 三栏（章节4 每个模块都要）
6. **章节2 只讲痛点**：方案和价值留给章节4 + 章节5（避免重复）
